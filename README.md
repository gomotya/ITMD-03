# ITMD-03

### Лабораторная работа №3: Разработка алгоритма выявления нарушений DRC на базе метода сканирующей прямой

### Цель работы:
Изучение алгоритма сканирующей (заметающей) прямой и получение практических навыков разработки логики для поиска геометрических нарушений проектных норм (DRC).

### Задача:
1. Использовать JSON-файл с топологией интегральной схемы из предыдущих лабораторных работ (содержащий искуственно внесенные нарушения). Название файлу задать `test_schema.json`.
2. Изучить предоставленный базовый "каркас" алгоритма сканирующей прямой (`sweepLine.py`). Каркас реализует движение вертикальной прямой по оси X слева направо и генерирует событие при встрече с каждой вершиной полигона.
3. Разработать логику выявления конкретного нарушения: **«Касание полигонов в одной точке»** (corner-to-corner touching). Полигоны не должны соприкасаться только одними углами без общих граней.
4. Выполнить соответствующие доработки базового каркаса в функции  внутри функции-коллбэка `detect_point_contact`, которая инжектируется в каркас. По результатам проверки формируется файл `student_predictions.json`.


### Формат выходного файла:
Структура отчета о нарушениях `student_predictions.json`:
```json
{
  "detailed_predictions":[
    {
      "source_element": 0,
      "target_element": 2,
      "violation_probability": 1.0,
      "status": "CRITICAL"
    }
  ]
}
```

### Состав отчёта:
1. Титульный лист;
2. Формулировка задания;
3. Доработанный файл `sweepLine.py`;
4. Изображение обрабатываемой топологии ИС;
5. JSON файл с найденными ошибками;
6. Вывод по работе.
